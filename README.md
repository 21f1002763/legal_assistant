# ⚖️ Legal Assistant

An AI-powered **Legal Assistant** that answers legal queries using **Retrieval-Augmented Generation (RAG)**.  
The application retrieves relevant legal provisions from a vector database and generates contextual responses using a Large Language Model.

It is designed to help users quickly understand **legal sections, acts, and provisions** by querying a curated legal knowledge base.

---

## 🚀 Features

- **Semantic Legal Search** using embeddings
- **AI-generated legal explanations**
- **Legal document indexing** with vector embeddings
- **Fast retrieval** with Chroma Vector Database
- **Retrieval-Augmented Generation (RAG)** pipeline
- **Vue.js frontend interface**
- **Flask REST API backend**

## ⚙️ Installation

### 1. Clone Repository

```bash
git clone https://github.com/yourusername/legal-assistant.git
cd legal-assistant
```

### 2. Install Backend Dependencies

```bash
cd backend
pip install -r requirements.txt
```

### 3. Start Ollama
#### Start the Ollama server:
```bash
ollama serve
```
#### Pull required models:
```bash
ollama pull nomic-embed-text
ollama pull mistral
```

### 4. Seed Legal Documents
#### This step chunks the legal documents and stores embeddings in the vector database.
```bash
python app/seed_documents.py
```

### 5. Run the Backend Server
```bash
python app.py
```
#### Backend will run on:
```bash
http://localhost:5000
```

### 6. Run the Frontend
```bash
cd frontend
npm install
npm run serve
```

#### Frontend will run on:
```bash
http://localhost:8080
```
---

## 📊 RAG Workflow
1. User submits a legal query
2. Query is converted into embeddings
3. ChromaDB retrieves relevant legal text chunks
4. Retrieved context is passed to the LLM
5. LLM generates a grounded response
